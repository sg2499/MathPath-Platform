- **Real bug found and fixed during Shailesh's local Step 4 visual review (2026-07-13):** Multiplication, Division, and Squares sections rendered as a bare "Multiplication = ?" / "Division = ?" / "Square = ?" with no operand numbers at all, only MCQ options -- confirmed via his screenshots. Root cause: `MathQuestionDisplay.tsx`'s `ExpressionQuestion`/`CompactExpressionQuestion` prefer `questionText` over building the expression from `operands`/`operators` when `questionText` is present (`AdminQuestionPreview.tsx` passes `q.question_text` straight through as `questionText`). `operands.py`'s `GenerateWholeNumberMultiplication`/`GenerateWholeNumberDivision`/`GenerateSquares` were setting `"question_text"` to a bare label ("Multiplication"/"Division"/"Square") instead of leaving it unset -- that label silently overrode the real expression. Fixed by removing `question_text` from those three functions so the frontend falls back to building `"3242 × 7 = ?"` / `"2236 ÷ 52 = ?"` / `"(45)² = ?"` from operands, exactly as BODMAS/Solve Equation already do correctly (they set `question_text` to the real expression, not a label). Checked every one of the 9 IM concept generators individually against the exact frontend display-mode logic (`MathQuestionDisplay.tsx`, `VerticalQuestion.tsx`) to confirm this was the only occurrence of the bug -- Add/Less, Skill Stacker, Concept Drill, and Answer Position all also set a generic `question_text` label, but their respective display components (`VerticalQuestion`, `CompactTwoColumnQuestion`, `PositionNumberTableQuestion`/`FirstNaturalNumberCardQuestion`) either ignore `questionText` entirely or use it only as an optional caption alongside the real operand table, so those were never at risk. Verified the fix by reimplementing the frontend's exact `FormatValue`/`BuildExpression` JS logic in Python and running it against all 730 Multiplication/Division/Squares questions generated across all 60 DPS -- 0 render issues. Full structural audit + idempotency + generation smoke test re-run clean after the fix (134 sections, 1,126 questions, 0 issues).
