# MathPath Platform QA Sweep — 2026-07-23

Full pass across Admin, Teacher, and Student logins on production (`math-path-platform.vercel.app`), done live with a real Chrome session. Goal: verify every metric/number shown actually matches what students have done, and catalog any discrepancies before further feature work. Primary test student: **Rohan Sen (MP-ST-008)**, teacher **Smita Singh**; secondary cross-check via teacher **Ashalatha Gupta**'s roster (24 students) and Admin's platform-wide views.

No test data was created or deleted during this sweep — read-only observation throughout.

---

## Confirmed bugs (reproducible, with exact numbers)

### 1. Student "Assessments" page is completely broken
`/student/assessments` (as Rohan Sen) shows a persistent error: **"SOMETHING WENT WRONG — Unable to load student assessments. Please refresh after signing in again, or ask Admin to verify assessment access."** Reproduced on two separate loads. The Module filter dropdown also has no options (`Choose Module` / `All Modules` only — no IM/MM/YLM entries), suggesting the same failed fetch feeds both. Sentry did capture something on this page load, confirming a real client-side exception, though the specific failing API call didn't surface in the network log (it appears to fire before the page is interactive).

Separately, Admin's platform-wide "Assessment Control" and the Teacher's "Assessment Tracker" both show **0 assigned assessments** across all students right now — so this may be compounded by nobody having any assessment assignments yet, but the hard error itself is a real, user-facing defect independent of that.

### 2. Mock Exam Leaderboard shows 0% for every rank below #3
On `/student/competition/leaderboard`, the podium (top 3) renders real scores correctly, but every row from rank #4 downward shows **"0" score / "0%" accuracy**, regardless of the student's actual performance. Confirmed on both leaderboard views:
- **Overall Journey** (MM-L1, all mocks): Rohan Sen sits at #7 with "0 0%" — despite his own Mock Performance Insights page (same session) showing a genuine 81% average across 6 completed mocks. Every other rank #4–#10 shows the same "0 0%" pattern.
- **Specific Exam** (MM_4 mock): Rank #4 (Ananya Dutta) shows "0 0%" while the top 3 show real percentages (85%, 65%, 50%).

This is a rendering/aggregation bug isolated to whatever component renders ranks below the podium — the underlying data clearly exists (the podium and the individual student's own stats page both have it correct).

### 3. Trophy Room badge counters are off by one vs. every other mock-count on the platform
Rohan Sen's Achievements page shows **"Super Competitor: 7/10"** and **"Legendary Competitor: 7/50"** (both "Complete N Mock Exams" badges). But every other view of the same data — his own Mock Exams list, his own Mock Performance Insights ("Total Attempts: 6"), and **Admin's platform-wide Competition Mock Tracker** (Rohan Sen: 8 Mocks, 6 Completed, 2 Pending, Avg Accuracy 81%) — consistently say **6 completed**, not 7. Three independent views agree on 6; only the badge-progress counter says 7. This points at the achievement-progress increment logic specifically, not the underlying mock records.

### 4. Mock exam average score/accuracy doesn't match the visible completed-mock scores
Rohan Sen's 6 completed mocks (visible in the Mock Exams table) score 65%, 90%, 90%, 80%, 100%, 80% — arithmetic mean **84.2%**. But both the Mock Exams summary card and the separate Mock Performance Insights page report **81%** average score/accuracy. A hidden 7th attempt at ~62% would reconcile the math, which would also tie back to finding #3 above (the badge counter's phantom 7th mock) — worth having engineering check whether there's an orphaned/soft-deleted attempt record still being folded into averages and badge counts but excluded from the visible list and "Total Attempts" tally.

### 5. Teacher "My Students" roster miscategorizes one student's DPS status
On Ashalatha Gupta's `/teacher/students` page, **Anaisha Agrawal** shows Assigned 2 / Cleared 1 / **Pending 1**. But the Practice Tracker (both the Overview stats and the per-student Student Review table) shows the same student as Assigned 2 / Cleared 1 / **Pending 0 / Needs Re-Attempt 1**. Same underlying fact (1 outstanding DPS), two different buckets on two different pages — a teacher glancing at "My Students" would think she just hasn't started it yet, when actually she scored low and needs to redo it. This is exactly the kind of number a teacher would act on incorrectly.

### 6. Teacher "Average Accuracy" headline stat is misleading
Ashalatha Gupta's Students page shows **"Average Accuracy: 8%"** for her 24 students. That number is computed by summing accuracy across *all 24 students* (including the 19 who have zero DPS assigned and thus no accuracy data, counted as 0) and dividing by 24. The 5 students who've actually done work average **37.6%** (or ~63% counting only those with a cleared attempt). An "8% average accuracy" headline is alarming and wrong in a way that would misdirect a teacher — it should either exclude students with no attempts from the denominator, or be labeled clearly as something else (e.g., "coverage-weighted").

### 7. Assessment Readiness detail card double-counts a re-attempt sheet as "cleared"
On the same page, for the same student, in the same load: Sakshi Agarwal's Assessment Readiness card (Teacher view) headlines **"2/80 cleared"** and **"CLEARED DPS: 2"**, but directly below it the filter-chip counts read **"Cleared (1) / Needs Re-Attempt (1)"**, and the actual sheet list confirms only DPS 1 is `Cleared` (90%) while DPS 3 is `Needs Re-Attempt` (60%). The top summary card is counting the re-attempt sheet as cleared; the detail breakdown two inches below it disagrees with itself.

### 8. "View Result" buttons on the Mock Exams table are unreliable via direct coordinate clicks
Not a confirmed backend bug, but worth a UI pass: clicking directly at the button's rendered coordinates repeatedly did nothing (no navigation, no console error, no network request) — only worked when targeting the exact accessibility-tree element. If real users are experiencing intermittent unresponsive clicks on this button (e.g., a hit-target/z-index issue), it would show up as "nothing happens when I click View Result."

## Verified working correctly

- **Student ↔ Admin mock exam counts match exactly.** Rohan Sen: 8 assigned / 6 completed / 2 pending / 81% avg, identical on both his own Mock Exams page and Admin's platform-wide Competition Mock Tracker. This is the most important cross-login check requested, and it holds.
- **Multiplication/division operator rendering is correct on live data.** Spot-checked a completed mock result (MM_26, 100%): "56 × 37 = ?" renders with the real `×` symbol and correct answer (2072) — matches the July 25 fix described in project history.
- **Teacher Practice Tracker is internally consistent.** Assigned (9) = Cleared (3) + Pending (4) + Needs Re-Attempt (2) reconciles exactly, and every other student's row matches the "My Students" page except the one Anaisha Agrawal discrepancy above.
- **Admin ↔ Teacher DPS/assigned counts match** for every other student checked (Sakshi Agarwal, Tanaya Patra, Aarav Mukherjee, Shailesh Gupta all consistent across Students page, Practice Tracker, and Assessment Readiness).
- **Teacher-to-student mapping is correct.** Confirmed via Admin's Student Directory that Rohan Sen belongs to teacher Smita Singh, not Ashalatha Gupta — his absence from Ashalatha's roster (despite competing on the same MM-L1 mock leaderboard as her students) is expected, not a bug.
- **The previously-flagged "stale IM L4 assessment data" issue appears resolved.** The live published "IM L4" assessment (84 questions) was spot-checked in Admin's Assessment Studio: division questions show real digit-pattern variety (7800÷78 = 4D/2D, 1728÷432 and 1608÷804 = 4D/3D) rather than the old always-2D×2D/4D÷2D default described in project history. Worth a fuller re-verification pass since only 3 questions were sampled, but the signal is positive — this was flagged in project docs as still awaiting a decision, and it looks like it may have already been handled.
- **Assessment Readiness top-level rollup is accurate.** Admin/Teacher "24 students reviewed, 0 ready, 24 not ready" is consistent with the fact that no student is anywhere close to the 70% practice-completion benchmark.

## Worth a look, not confirmed as bugs

- **Parent Report delivery has 1 failed send** (Shailesh Gupta / MP-ST-001, YLM-L1, dated 27/5/2026) sitting in Admin → Assessment Control → Parent Reports with no visible retry action taken. Not urgent, but a silently-failed parent communication is worth someone's attention.
- **Dashboard "Last Activity" date format is ambiguous.** Rohan Sen's Progress page showed "LAST ACTIVITY: 9/6/2026, 2:23:07 PM" — if that's M/D/Y it's a date in the future relative to today (23 Jul 2026), which shouldn't be possible; if D/M/Y it's fine (9 June). Worth confirming the app is consistent about which format it uses everywhere, since a mixed format would silently corrupt any date-based sorting or "stale data" logic.
- **No assessments are assigned to anyone on the platform right now** (0 across Admin, and every teacher/student checked). May just reflect where the rollout currently stands, but flagging in case it's not intentional.
